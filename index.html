At some point the documentation stops being a guide and becomes an archaeological artifact: an architecture diagram from three reorganizations ago, setup instructions that have never worked, a Getting Started guide that starts with “Ask Rahul.”

The failure isn't laziness — it's that no human is the right scheduler for this job. Docs drift the moment attention moves on. So LivingDocs assumes a machine runs it: cron, a GitHub Action, or a git hook. It finds its own work, fixes it, verifies the fix, and stops. Zero findings is the stop condition — the loop can't run away.

And because drift detection is graph math, not model vibes, the “is this doc wrong?” step is free, offline, and deterministic. Same graph in, same answer out. Every time.

the whole cli

      
Eight commands. Two that matter.

      

        
livingdocs init
Scaffold config + managed doc sections in any repo.
offline

        
livingdocs analyze
Build the knowledge graph from your code. Incremental — keyed off the git diff.
offline

        
livingdocs check
Report drift. Exit 0 clean · 1 drift · 2 error. The CI gate.
offline · hero

        
livingdocs update
Detect → fix drifted sections → verify → PR. The scheduled entrypoint.
calls openai

        
livingdocs sync
Regenerate every managed section from the graph.
calls openai

        
livingdocs watch
Debounced re-check on file changes while you develop.
offline

        
livingdocs explain
Grounded explanation of a symbol. The garnish, not the meal.
calls openai

        
livingdocs review
Architecture review: circular deps, large modules, tight coupling.
offline

faq

      
Reasonable questions, honest answers.

      

        

          Isn't this just another “chat with your repo” tool?
          
No. You can ask it things, but that's the garnish. The main course is drift detection on a schedule: it tells you — and fixes — which docs have quietly become false, every week, without being asked. A chat answer disappears when you close the tab. Docs don't.

        

        

          Does my code leave my machine?
          
Drift detection never does — it's local graph math, no API call, no key needed. For the writing step, LivingDocs sends a structured summary (entities, dependencies, routes) to OpenAI. It sends the graph, not your raw files.

        

        

          Will it push weird AI commits straight to main while I sleep?
          
No. It opens a pull request. A human merges it. Your main branch is safe from its enthusiasm.

        

        

          What languages does it understand?
          
TypeScript and JavaScript at launch, including Express route extraction. Java and Python are next. The parser layer is pluggable — each new language is an addition, not a rewrite.

        

        

          Is this a VS Code extension?
          
No. It's a CLI. It lives in your terminal and your CI, not your editor. An editor client may come later — the CLI is the product.

        

        

          Does it judge my code?
          
Officially, no. Unofficially, livingdocs review has opinions about your circular dependencies.
